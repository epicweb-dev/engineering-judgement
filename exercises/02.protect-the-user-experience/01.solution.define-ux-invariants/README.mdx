# Define UX Invariants

👨‍💼 Nice work. Before touching implementation, you identified the user journeys
that matter most and made the non-negotiable experience constraints explicit.

🦉 This is the point of the exercise: strong engineers do not wait for regressions
to show up in production before deciding what was important. They define the
critical flows, invariants, and degradation signals up front so future tradeoffs
can be judged against something concrete.

In the next step, you'll use those invariants as the guardrails for a product
change.
