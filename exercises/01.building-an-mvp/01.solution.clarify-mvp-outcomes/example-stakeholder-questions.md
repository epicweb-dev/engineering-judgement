# Example Stakeholder Questions

Use these as quality examples, not a required script.

## Use a realistic MVP stakeholder set

For an MVP, you usually do not need a dozen interviews. A strong early-stage
set is:

- 💼 Berry the Business Owner (business objective and launch decision owner)
- 👤 Una the User (real workflow pain and usability needs)
- Engineering counterpart (feasibility and delivery constraints)
- Optional fourth stakeholder when required (legal/compliance or support/ops)

## Decision-oriented questions by stakeholder

### 💼 Berry the Business Owner

1. What single business outcome should this MVP improve in the first 30 days?
2. What is the minimum launch metric that tells us this MVP is viable?
3. Which user segment is in scope for MVP, and which segments are explicitly out?

### 👤 Una the User

4. What is the exact scheduling pain in your current workflow?
5. Which step in the current process causes the most drop-off or delays?
6. What would make you trust this MVP enough to use it for real scheduling?

### Engineering counterpart

7. What technical constraints should shape MVP scope (time, systems, team size)?
8. Which integration can be deferred without invalidating MVP viability?
9. What failure mode is most likely in v1, and what mitigation should we include?

### Optional legal/compliance or support/ops stakeholder

10. Are there data retention, privacy, or support constraints we must satisfy at launch?

These are strong because each answer drives a concrete decision on scope,
architecture, sequencing, or launch criteria.

## Weak questions (anti-pattern examples)

- "Can you tell me more about the app?"
- "Do users like simple interfaces?"
- "Should we make this scalable?"
- "Any other requirements?"

These are weak because they are too broad and do not force concrete decisions,
tradeoffs, or acceptance criteria.

## Self-check rubric

- **Decision impact:** Does each answer change a real decision?
- **Risk coverage:** Did you ask about the biggest ways this could fail?
- **Testability:** Can success be measured with observable criteria?
- **Scope control:** Did you separate MVP requirements from future ideas?
