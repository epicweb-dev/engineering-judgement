# Constrain the Response

👨‍💼 Nice work. Instead of letting a large request pull the product into a much
broader shape, you constrained the response to the smallest move that might
solve the real problem.

🦉 This step matters because teams often do the hard part first, qualify that a
request is too broad, and then quietly lose the judgment by implementing too
much anyway. Strong engineers protect the product by making the acceptable
scope explicit before any build work begins.

You now have a response-constraint brief that defines what the team could do
next, what should stay out of scope, and what evidence would need to change
before a larger solution deserves reconsideration.
