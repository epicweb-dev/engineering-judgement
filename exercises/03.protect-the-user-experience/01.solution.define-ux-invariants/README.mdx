# Define UX Invariants

👨‍💼 Nice work. Before implementing the newly justified host-login work, you
identified the user journeys that matter most and made the non-negotiable
experience constraints explicit.

🦉 This is the point of the exercise: strong engineers do not wait for login to
ship before deciding what was important. They define the critical flows,
invariants, and degradation signals up front so implementation tradeoffs can be
judged against something concrete.

In the next step, you'll turn those invariants into an implementation plan that
aims to add host login without regressing the product's most important UX.
